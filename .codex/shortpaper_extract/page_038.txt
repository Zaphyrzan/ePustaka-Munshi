 
3.3.6  Review  
The  end  of  each  sprint  is  followed  by  a  Sprint  Review  for  the  stakeholder.  This  is  a  
team
 
activity
 
wherein
 
the
 
developer
 
shows
 
the
 
implemented
 
functionality.
 
This
 
is
 
a
 
direct
 
feedback
 
session.
 
Findings  from  the  review  process  are  valuable,  such  as,  if  the  Librarian  thinks  that  
the
 
OCR
 
validation
 
form
 
is
 
cluttered,
 
this
 
input
 
is
 
then
 
incorporated
 
into
 
a
 
new
 
task
 
for
 
the
 
next
 
sprint.
 
This
 
process
 
allows
 
for
 
a
 
positive
 
outcome
 
to
 
be
 
gained
 
from
 
the
 
final
 
production.
