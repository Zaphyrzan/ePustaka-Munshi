 
4.3  Project  Design  
This  section  translates  the  requirements  into  a  technical  structure.  The  ePustaka  
Munshi
 
system
 
utilizes
 
a
 
modern
 
web
 
application
 
architecture
 
designed
 
for
 
scalability
 
and
 
maintainability.
 
4.3.1  System  Architecture  
ePustaka  Munshi  is  based  on  Three  Tier  Architecture  modified  to  suit  a  web-based  
environment.
 
This
 
helps
 
in
 
keeping
 
the
 
architecture
 
concerned-free,
 
scalable,
 
and
 
maintenance-friendly
 
and
 
makes
 
use
 
of
 
cloud
 
technologies
 
to
 
minimize
 
usage
 
of
 
server
 
space.
 
1.  View  Layer  (Client-Side  Frontend):  
Built
 
with
 
React.js,
 
this
 
is
 
responsible
 
for
 
user
 
interface
 
rendering.
 
It
 
is
 
broken
 
down
 
into
 
specific
 
UI
 
packages
 
for
 
the
 
core
 
system
 
modules
 
of
 
User
 
Management,
 
Book
 
Catalog,
 
Ledger
 
Digitization,
 
and
 
Circulation.
 
This
 
system
 
will
 
be
 
executed
 
on
 
the
 
client
 
browser
 
environment,
 
capturing
 
keyboard
 
and
 
USB
 
barcode
 
scanner
 
inputs,
 
and
 
displaying
 
real-time
 
feedback
 
for
 
the
 
user.
 2.  Domain  Layer  (Application  Logic):  
It
 
fills
 
the
 
gap
 
between
 
“user
 
interface”
 
and
 
“data.”
 
It
 
has
 
controllers
 
for
 
the
 
“business
 
logic”
 
such
 
as
 
“Auth
 
Manager,”
 
“Catalog
 
Manager,”
 
“Circulation
 
Manager.”
 
Most
 
importantly,
 
it
 
houses
 
the
 
“OCRProcessor”
 
logic
 
that
 
serves
 
as
 
the
 
link
 
between
 
the
 
“Upload
 
interface”
 
and
 
the
 
external
 
“OCR
 
engine,”
 
making
 
sure
 
that
 
“raw
 
data”
 
has
 
been
 
parsed
 
and
 
cleaned
 
for
 
verification
 
purposes.
 3.  Data  Access  Layer  (Backend  &  Database):  
The
 
system
 
uses
 
Supabase
 
as
 
a
 
Backend-as-a-Service
 
(BaaS).
 
In
 
this
 
layer,
 
the
 
storage
 
is
 
managed
 
through
 
a
 
PostgreSQL
 
database.
 
In
 
the
 
Data
 
Access
 
Layer,
 
all
 
the
 
CRUD
 
(Create,
 
Read,
 
Update,
 
and
 
Delete)
 
functions
 
are
 
managed,
 
and
 
the
 
relationships
 
between
 
users,
 
books,
 
and
 
transactions
 
are
 
upheld.
 
Additionally,
 
authentication
 
tokens
 
and
 
Row
 
Level
 
Security
 
policies
 
are
 
managed
 
to
 
ensure
 
the
 
safeguarding
 
of
 
sensitive
 
data
 
for
 
students.
